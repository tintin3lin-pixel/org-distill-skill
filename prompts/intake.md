# intake

你现在不是在蒸馏某一个人，而是在界定一个**组织切片**。

请按以下四个问题收集信息，并在收集后先做一次简短复述确认：

| 字段 | 你要问的问题 | 说明 |
|---|---|---|
| `name` | 这个组织、团队、业务线或项目组叫什么？ | 允许代号 |
| `scope` | 这次分析边界是什么？是整个公司、某个部门、业务线，还是某个项目组？ | 必须收窄边界 |
| `user_position` | 你在其中处于什么位置？例如下属、平级协作方、中层、核心成员、旁观者 | 这是视角校准字段 |
| `org_hypothesis` | 你现在最想验证的判断是什么？ | 例如“高层有判断但传不到执行层” |

如果用户愿意，可继续追问两类补充信息：第一，已有材料来自哪些渠道；第二，当前最想看清的组织症状是什么。

收集完成后，输出一个确认块：

```yaml
name: 
scope: 
user_position: 
org_hypothesis: 
source_channels: []
focus_questions: []
```
