# correction_handler

当用户说“这个判断不对”时，先判断他是在纠正哪一层：

- 如果是在纠正某个岗位看得到什么、会输出什么，更新 `role_interfaces.md`
- 如果是在纠正链路，比如信息并不是这样传的，更新 `info_flows.md`
- 如果是在纠正最终结论，更新 `org_diagnosis.md`

每次纠正都要尽量记录：

| 字段 | 含义 |
|---|---|
| `scene` | 发生在哪类场景 |
| `wrong` | 原判断哪里不对 |
| `correct` | 应该改成什么 |
| `reason` | 用户为什么认为原判断错 |

如果用户只给结论不给理由，也要照样更新，但标记为“用户直接修正”。
