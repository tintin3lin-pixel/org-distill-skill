# 参考资料总览

`references/` 目录面向已经准备继续二次开发、扩展分析逻辑或对接结果消费层的开发者。它不是第一次上手项目的入口；如果你还没试跑过样例，请先回到 [`README.md`](../README.md) 和 [`QUICKSTART.md`](../QUICKSTART.md)。

## 一、这个目录解决什么问题

当你继续往下开发时，通常会遇到三类问题：

| 问题 | 建议阅读文档 |
| --- | --- |
| 输入目录和产物字段具体怎么约定 | [`api_reference.md`](./api_reference.md) |
| 什么样的材料算有效组织信号 | [`org_signal_taxonomy.md`](./org_signal_taxonomy.md) |
| 不同证据类型应该如何区分轻重 | [`evidence_weighting.md`](./evidence_weighting.md) |

## 二、适合谁读

| 角色 | 为什么值得读 |
| --- | --- |
| Agent 集成开发者 | 需要理解字段协议和结果消费边界 |
| 做二次分析的人 | 需要知道哪些证据更应被优先相信 |
| 想改推断逻辑的人 | 需要理解当前方法论如何定义组织信号 |

## 三、不适合放在这里的内容

为了保持公开仓库结构清晰，这个目录不再放产品介绍、运行指南或内部协作文档。它只保留**参考型资料**：协议、分类法、权重说明和其他面向开发的背景材料。
